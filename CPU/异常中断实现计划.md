# CPU 异常/中断功能实现计划

## 目标

在现有 `CPU/` 单周期 MIPS 工程基础上,按照《计算机原理与设计 Verilog HDL版》第6章实现:
- 3 种内部异常:算术溢出、未实现指令、系统调用 syscall
- 1 个外部中断(查询方式,单源)
- 4 条新指令:mfc0、mtc0、syscall、eret
- 3 个 CP0 寄存器:Status(#12)、Cause(#13)、EPC(#14)
- 配套测试程序,覆盖全部 4 种异常/中断场景

## 关键常量(取自教材)

| 项目 | 取值 |
|---|---|
| 异常/中断总入口 | `EXC_BASE = 0x00000008` |
| 复位入口 | `0x00000000` |
| CP0 Status | 寄存器号 12 |
| CP0 Cause | 寄存器号 13 |
| CP0 EPC | 寄存器号 14 |
| ExcCode(Cause[3:2]) | 00=外部中断, 01=syscall, 10=未实现指令, 11=溢出 |
| Status IM[3:0] | sta[3]=溢出, sta[2]=未实现指令, sta[1]=syscall, sta[0]=外部中断 |
| mfc0 | op=0x10, rs=0x00 |
| mtc0 | op=0x10, rs=0x04 |
| syscall | op=0x00, func=0x0c |
| eret | op=0x10, rs=0x10, func=0x18 |

## 数据通路改动

1. ALU 增加 `v`(overflow)输出,仅 `add/sub/addi` 触发。
2. 新增 3 个 CP0 寄存器模块,带写使能 `wsta/wcau/wepc`。
3. PC 选择新增 `selpc[1:0]`:00=正常 npc,01=eret 返回 EPC,10=异常入口 EXC_BASE。
4. mfc0 写回数据选择:在 `{alu, mem}` 基础上增加 `{sta, cau, epc}` 选择。
5. 控制单元新增异常/中断译码及控制信号生成。

## 板上演示要点

- `KEY[0]` 复位,`KEY[1]` 触发外部中断。
- 用 `SW[1:0]=01` 观察 PC:
  - 溢出:PC=0x84 的 `add` 触发 → 0x08 → 0x68
  - syscall:PC=0x8c 触发 → 0x08 → 0x3c
  - 未实现指令:PC=0x94 的 `div` 触发 → 0x08 → 0x54
  - 外部中断:在 `loop` 中按 `KEY[1]` → 0x08 → 0x30 → 返回继续循环

## 注意事项

- 保留 `CPU/` 已修正的 16 位分支偏移,不采用教材/参考代码中的 `imm[13:0]`。
- 不引入教材的 `sc_interrupt` 包装层,直接在现有顶层例化结构上做。
- 教材测试程序使用 nop 填充跳转/分支延迟,本工程单周期无延迟槽,nop 仅作安全填充。
