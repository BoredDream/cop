# 单周期 MIPS CPU —— 板上演示说明

> 适用工程:`CPU/DE1_SOC_golden_top.qpf`(Quartus II 14.0,DE1-SoC,Cyclone V `5CSEMA5F31C6`)
> 配套:指令程序见 `sccpu/scinstmem.v`、数据见 `sccpu/scdatamem.v`;仿真测试见 `sccpu/tb_intr.v`。

---

## 0. 一句话原理

这是一个**带异常/中断处理的单周期 MIPS CPU**:每来一个时钟上升沿执行**一条**指令。指令存储器里烧的是教材第6章测试程序,会依次触发**算术溢出、系统调用、未实现指令、外部中断**四类异常/中断,并通过**数码管 + LED** 实时显示 PC 与运算结果。

---

## 1. 演示前准备

1. Quartus 打开 `CPU/DE1_SOC_golden_top.qpf`。
2. `Processing → Start Compilation` 全量编译(约 5 分钟),生成 `output_files/DE1_SOC_golden_top.sof`。
3. `Tools → Programmer` 烧录 `.sof` 到开发板(选 `5CSEMA5` 那行 → Change File → 勾 Program/Configure → Start)。
4. 烧完即开始运行。

---

## 2. 操作面板速查(本工程的开关/按键定义)

| 控件 | 作用 |
|---|---|
| **`KEY[0]`** | 复位:**按下**复位归零,**松开**开始运行 |
| **`KEY[1]`** | 外部中断请求 `intr`(**按下**产生中断) |
| **`SW[3]`** | CPU 时钟速度:`0`=快(10MHz,瞬间跑完)、`1`=慢(约 1Hz,**逐拍肉眼可见**) |
| **`SW[1:0]`** | 数码管显示**哪一路信号**(见下表) |
| **6 位数码管 HEX5..0** | 显示 `SW[1:0]` 选中的 24 位信号(十六进制) |
| **`LEDR[5:0]`** | 当前指令的 `opcode` |
| **`LEDR[6]`** | 中断确认 `inta`(进入外部中断时亮一拍) |
| **`LEDR[7]`** | **程序结束指示**:PC 停在 `0xC0` 时点亮 |
| **`LEDR[8]`** | 当前速度档(=`SW[3]`) |
| **`LEDR[9]`** | 复位状态(=`KEY[0]`) |

### `SW[1:0]` 四种显示模式
| `SW[1:0]` | 数码管显示 | 用途 |
|:---:|---|---|
| `00` | `{opcode, func}` | 看**指令类型**(操作码+功能码) |
| `01` | **PC**(程序计数器) | 看**程序走到哪一步/是否进入异常处理**(最直观) |
| `10` | 完整 **32 位指令码** | 看**取到的机器码** |
| `11` | **ALU 输出** | 看**运算/访存的结果** |

## 3. 推荐演示流程(照着做)

### 步骤 ①:慢速 + 看 PC(主证据)
1. 拨 **`SW[3]=1`**(慢钟),**`SW[1:0]=01`**(显示 PC)。
2. 按一下 **`KEY[0]`** 再松开 → CPU 从头开始。
3. 数码管会**逐拍**显示 PC:
   ```
   000000 → 000074 → 000078 → 00007C → 000080 → 000084 → 000008 → 000068 → 00006C → 000040 → 000044 → 000048 → 00004C → 000088 → 00008C → 000008 → 00003C → ... → 000090 → 000094 → 000008 → 000054 → ... → 000098 → 00009C → 0000A0 → 0000A4 → 0000A8 → ...
   ```
   讲解:
   - `000000` 复位后 `j start` 跳到 `000074`。
   - `000078` 用 `mtc0` 打开中断屏蔽位(`IM[3:0]=1111`)。
   - `000084` 的 `add` 产生**溢出异常** → PC 跳到 `000008`(异常总入口) → `000068`(溢出处理程序) → EPC+4 后返回。
   - `00008C` 的 `syscall` 触发异常 → 入口 `00003C`。
   - `000094` 的 `div`(未实现指令)触发异常 → 入口 `000054`。
   - 进入 `loop` 循环后,按 **`KEY[1]`** 触发**外部中断** → PC 跳到 `000008` → `000030`,`LEDR[6]` 点亮一拍,返回后继续循环。
   - 最终停在 **`0000C0`** 的 `j finish` 死循环,`LEDR[7]` 点亮。
4. **`LEDR[7]` 此时点亮** → 程序正确执行到结尾。✅ 这是"运行正确"的核心判据。

### 步骤 ②:切换四种显示,逐一展示
保持慢钟,依次拨 `SW[1:0]`:
- **`00`(opcode+func)**:数码管随每条指令变化,可看到 `mfc0`/`mtc0`/`syscall`/`eret` 等新指令的 opcode。
- **`10`(指令码)**:显示当前取到的 32 位机器码低位,可与 `scinstmem.v` 里的十六进制逐条对上(如 `401a6800`=`mfc0`、`40886000`=`mtc0`、`42000018`=`eret`)。
- **`11`(ALU 输出)**:显示每拍运算结果,能看到 `add` 溢出后 `aluout=80000001`、EPC+4 计算等。

### 步骤 ③(可选):外部中断演示
在 `loop` 循环运行期间(PC 在 `0000A8` 附近反复),按一下 **`KEY[1]`**:
- 下一拍 PC 跳到 `000008`,同时 **`LEDR[6]` 亮**(中断确认 `inta`)。
- 随后进入 `000030` 的中断处理程序,执行 `eret` 后返回 `loop`。
- 若 `KEY[1]` 一直按住,会因电平敏感而反复进入中断,所以**点按一下**即可。

### 步骤 ④(可选):快速模式
拨 **`SW[3]=0`**(快钟),按 `KEY[0]` 复位 → CPU 瞬间跑完,数码管/LED 直接停在终态(PC=`0000C0`,`LEDR[7]` 亮)。用来说明"实际 10MHz 下一闪而过,慢钟只是为了演示观察"。

## 4. 这段程序在做什么(讲解素材)

教材第6章测试程序按顺序验证 4 种异常/中断:

1. **开中断**:用 `addi r8, r0, 0xf` 把 `0xF` 写入 `Status` 寄存器,使能 `IM[3:0]`。
2. **溢出异常**:从数据存储器读出 `0x00000002` 和 `0x7FFFFFFF`,`add` 后溢出。
3. **系统调用异常**:执行 `syscall`。
4. **未实现指令异常**:执行 `div`(本 CPU 未实现)。
5. **外部中断**:在 `loop` 累加数据时,由 `KEY[1]` 产生中断请求。

异常/中断统一入口为 `0x00000008`,由软件读取 `Cause` 寄存器散转到各自处理程序:
- 外部中断 → `0x30`
- syscall → `0x3C`
- 未实现指令 → `0x54`
- 溢出 → `0x68`

## 5. 新增硬件与指令(答辩要点)

**新增指令**: `mfc0`、`mtc0`、`syscall`、`eret`

**新增 CP0 寄存器**:
- `Status(#12)`:中断屏蔽位 `IM[3:0]`
- `Cause(#13)`:异常原因 `ExcCode`
- `EPC(#14)`:返回地址

**异常/中断处理流程**:
1. `EPC ← pc`(异常) 或 `EPC ← npc`(中断)
2. `Cause.ExcCode ← 原因`
3. `Status ← Status << 4`(自动关中断)
4. `PC ← 0x00000008`

**返回**: `eret` 时 `PC ← EPC`,`Status ← Status >> 4`。

## 6. 故障排查

| 现象 | 处理 |
|---|---|
| 数码管不动 / 全灭 | 确认 `KEY[0]` 已松开(按住=一直复位);确认已烧录 `.sof` |
| 跑太快看不清 | 拨 `SW[3]=1` 进慢钟 |
| `LEDR[7]` 不亮 | 说明没停在 `0xC0`,检查是否烧的是本工程、`scinstmem.v`/`scdatamem.v` 是否被改过 |
| 按 `KEY[1]` 无中断 | 确认程序已执行到 `loop`(PC 在 `0xA8` 附近);确认 `KEY[1]` 是点按不是长按 |
| Programmer 检测不到板 | Hardware Setup 选 `DE-SoC [USB-1]`,检查 USB Blaster 驱动 |
| SignalTap 抓不到 | 必须**先完全编译再烧录**(SignalTap 嵌在比特流里),再按 `KEY[0]` 触发 |
