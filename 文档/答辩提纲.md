## 1. 指令集分析

> 讲法:**先总体,后样例**——先给全景,再挑几条代表指令逐条剖;其余指令都是这几类的变体。

### 1.1 总体介绍(全景)

- 三种指令格式:R 型(op+rs+rt+rd+sa+func)、I 型(op+rs+rt+imm)、J 型(op+address)。
- 24 条指令一览表(20 基本 + 4 异常),按类型分组。
- 强调:这是**分析/选定**讲义与教材给定的 MIPS 子集,不是自创 ISA。
- 🔁 复用现有 PPT「20 条 MIPS 指令」整张表。

### 1.2 代表指令逐条剖(4 条 archetype = 全场主线)

挑选标准:每条"点亮"数据通路里一类不同行为,第 3/4 节会反复用到。

| 代表指令 | 类型 | 为什么选它(代表了什么) |
| --- | --- | --- |
| `add` | R 型 | 最基础:寄存器→ALU→写回;埋下"溢出异常"伏笔(接第 5 节) |
| `lw`  | I 型 | 数据通路**最长**:取指→译码→ALU 算地址→访存→写回;立即数符号扩展 |
| `beq` | I 型 | ALU 做减法**只取 `z` 标志**,不要差值;PC 相对跳转,引出 `pcsource` |
| `jal` | J 型 | 写返回地址到 **r31**,引出写回端多路选择与 PC+4 |

- 每条配 RTL + 一句"特殊在哪"。🔁 RTL 复用现有 PPT 手写 RTL 各页。
- 收尾一句:`sub/and/or/xor`≈`add` 变体,`sw/addi/andi…`≈`lw` 变体,`bne`≈`beq`,`j`≈`jal`——**20 条 = 4 类原型的排列组合**。

📁 对应文件:`sccpu/scinstmem.v`(指令编码),命中信号在 `sccpu/sccu_dataflow.v`。

## 2. ALU 设计

> 讲法:**先整体介绍 ALU**(角色 + 9 运算 + 结构),再点 1~2 个代表运算细看。

### 2.1 总体介绍

- 角色:CPU 唯一的"计算核心";端口 `alu(a, b, aluc, r, z, v)`,4 位 `aluc[3:0]` 选运算。
- **9 种运算**及编码:ADD `0000`、SUB `0100`、AND `0001`、OR `0101`、XOR `0010`、LUI `0110`、SLL `0011`、SRL `0111`、SRA `1111`。
- 结构思想:**并行算出全部候选结果,再用 `mux4x32` 按 `aluc[1:0]` 四选一**(addsub32 / 与或 / 异或·LUI / shift)。
- 🔁 复用现有 PPT「ALU 设计」结构框图。

### 2.2 代表运算细看(1~2 个)

- `add`/`sub`:基于 `addsub32`,顺带讲**溢出标志 `v`**(同号相加/异号相减后符号翻转)——它是第 5 节"溢出异常"的硬件来源。
- `shift`:结构上的"异类",操作数来自 `a[4:0]`(移位量),还要 `right`/`arith` 两个控制位区分逻辑/算术、左/右移。
- 另:`z = ~|r` 零标志供 `beq/bne` 判分支。

📁 对应文件:`sccpu/alu.v`(依赖 `addsub32.v`/`shift.v`/`mux4x32.v`)。

## 3. 数据通路设计  *(数据通路在控制器之前)*

> 讲法:**先给全景框图(一张地图),再沿第 1 节那几条代表指令"走一遍"**。框图静态看很难懂,"跟着一条指令走"就变成可跟随的故事——这是讲数据通路最有效的方式。

### 3.1 全景框图

- 单周期 = **取指 → 译码 → 执行 → 访存 → 写回**,一个时钟周期内全部完成(无流水线)。
- 关键部件:PC `dff32`、指令 ROM `scinstmem`、寄存器堆 `regfile`(2 读 1 写,r0 恒 0)、ALU 输入选择 `alua/alub`、数据 RAM `scdatamem`、写回三级 mux、下一 PC 选择 `pcsource`。

🖼️ **【待插入图片】总数据通路框图** — 来源:Quartus **RTL Viewer** 对 `sccpu_dataflow` 截图,或 `实验设计报告.md` 第 2.1 节框图。

### 3.2 沿代表指令"点亮"路径(主线,呼应第 1 节)

在同一张框图上逐条高亮活跃路径:

- `add`:regfile 读 rs/rt → ALU(ADD)→ 写回 rd。**最短主干**。
- `lw` :regfile 读 rs → ALU 算 `rs+符号扩展(imm)`(地址)→ 数据 RAM 读 → 写回 rt。**走完全程**(多了访存)。
- `beq`:regfile 读 rs/rt → ALU(SUB)出 `z` → `pcsource` 选分支目标 `(PC+4)+offset`。**不写寄存器,改的是 PC**。

> 三条走完,数据通路每条支路(ALU 运算 / 访存 / 分支改 PC / 写回)都点亮过了。

📁 对应文件:`sccpu/sccpu_dataflow.v`。

## 4. 控制器设计  *(控制器在数据通路之后)*

> 讲法:**职责 → 译码方法 → 用同一批代表指令看控制信号 → 核心论点(9→20)**。承接第 3 节:数据通路每个 mux,都由这里发的信号选择。

### 4.1 职责与译码方法

- **纯组合逻辑**:由 `op`/`func` 先还原每条指令的命中信号 `i_xxx`,再合成所有控制信号。
- 主要控制信号:`wreg/regrt/m2reg/wmem/aluimm/shift/sext/aluc[3:0]/pcsource[1:0]/jal`。

### 4.2 用代表指令看"信号怎么配"(继续用第 1/3 节那几条)

| 指令 | 关键置位信号 | 含义 |
| --- | --- | --- |
| `add` | `wreg, aluc=ADD` | 写 rd |
| `lw`  | `wreg, regrt, m2reg, aluimm, sext, aluc=ADD` | 算地址 + 访存写回 rt |
| `beq` | `sext, aluc=SUB, pcsource←branch_taken` | 取 z 判分支 |
| `jal` | `wreg, jal(写 r31), pcsource=11` | 存返回地址 + 跳转 |

### 4.3 ⭐ 核心论点:9 种 ALU 运算如何扩展成 20 条指令

- 复用:多条指令落到同一种运算(add/addi/lw/sw 都用 ADD;beq/bne 用 SUB 只取 `z`;and/andi 用 AND…)。
- 区分:靠正交控制信号(`aluimm/sext/m2reg/wmem/pcsource/shift`)裁剪同一 ALU 结果的去向。
- 一句话:**指令数 = ALU 运算 × 数据通路其它控制信号的组合**。

## 5. 异常/中断设计  *(本组自定提升,讲义未要求)*

> 讲法:**为什么 → 机制三段(检测/响应/处理返回)→ 完整追踪一次异常 → 新增电路**。和第 3 节一样用"追踪一例"把抽象机制讲活。

### 5.1 为什么 + 总体

- 3 内部异常:算术溢出、`syscall`、未实现指令;1 外部中断:`KEY[1]`(单源、查询)。
- 4 新指令 `mfc0/mtc0/syscall/eret`;3 个 CP0 寄存器 `Status(#12)/Cause(#13)/EPC(#14)`。
- 统一入口 `EXC_BASE = 0x00000008`。

### 5.2 机制三段

1. **检测(+屏蔽)**:4 个事件经 `Status.IM[3:0]` 屏蔽后或合成总异常 `exc`;`Cause.ExcCode` 记录原因。
2. **硬件响应(触发当拍)**:`EPC←pc/npc` → 写 `Cause` → `Status<<4`(自动关中断、防嵌套)→ `PC←0x08`。
3. **软件处理 + 返回**:从 `0x08` 读 `Cause` 查**散转表** → `jr` 到各处理程序 → `eret`(`PC←EPC`、`Status>>4` 自动开中断)。

### 5.3 完整追踪一例(主线手法)

- 溢出:`0x84 add` 溢出 → 硬件跳 `0x08` → 读 Cause 散转到溢出 entry `0x68` → 处理(`EPC←EPC+4` 跳过出错指令)→ `eret` 返回。
- 外部中断同理:循环中按 `KEY[1]` → `inta` 亮 → `EPC←npc` → 跳 `0x08` → 中断 entry `0x30`。

🖼️ **【待插入图片】异常/中断处理流程图** — 来源:`实验设计报告.md` 第 3.5 节。

### 5.4 新增电路

- CP0 三寄存器(`dffe32`,写使能 `wsta/wcau/wepc`)。
- PC 最终选择 `selpc` 四选一(正常 npc / eret→EPC / 异常→0x08)。
- `mfc0` 读出选择 `fromc0`(Status/Cause/EPC)。

🖼️ **【待插入图片】数据通路新增电路局部图** — 来源:第 3.6 节。

📁 对应文件:`sccpu/sccpu_dataflow.v`(CP0/PC 选择)+ `sccpu/sccu_dataflow.v`(异常检测/优先级)。

## 7. 问答

- **判据回顾**:程序停在 `0xC0` 且 `LEDR[7]` 亮 = 完全正确;按 `KEY[1]` 见 `LEDR[6]` 闪 = 中断正确。
- 预设可能被问到的问题(内部准备,不一定上 PPT):
  - 9 种 ALU 运算怎么支撑 20 条指令?(见第 4 节核心论点)
  - 为什么 `beq/bne` 不需要 ALU 算差值?(只用减法的 `z` 标志)
  - 异常入口为什么要把 `Status` 左移关中断?(防异常嵌套)
  - `syscall`/溢出处理后为何 `EPC←EPC+4`?(跳过出错/陷入指令)
  - 单周期相比多周期/流水线的取舍?(实现简单、时钟受最慢指令限制)
