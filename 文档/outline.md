## 第4页 · 指令集分析（4 条代表指令）

- 挑选标准：每条"点亮"数据通路里一类不同行为，第 3、4 节反复用到

| 代表指令 | 类型 | 代表了什么 |
|---|---|---|
| `add` | R | 最基础：寄存器→ALU→写回；埋下溢出异常伏笔（接第 5 节） |
| `lw` | I | 数据通路最长：取指→译码→算地址→访存→写回；立即数符号扩展 |
| `beq` | I | ALU 减法**只取 `z` 标志**，不要差值；PC 相对跳转，引出 `pcsource` |
| `jal` | J | 写返回地址到 **r31**；引出写回端多路选择与 PC+4 |

- 收口：`sub/and/or/xor`≈`add` 变体，`sw/addi…`≈`lw` 变体，`bne`≈`beq`，`j`≈`jal`——**20 条 = 4 类原型的排列组合**

## 第5页 · ALU 设计（总体）

- 角色：CPU 唯一的"计算核心"；端口 `alu(a, b, aluc, r, z, v)`，4 位 `aluc[3:0]` 选运算
- 结构思想：**并行算出全部候选结果，再用 `mux4x32` 按 `aluc[1:0]` 四选一**（addsub32 / 与或 / 异或·LUI / shift）

| 运算 | 编码 | 运算 | 编码 |
|---|---|---|---|
| ADD | 0000 | LUI | 0110 |
| SUB | 0100 | SLL | 0011 |
| AND | 0001 | SRL | 0111 |
| OR | 0101 | SRA | 1111 |
| XOR | 0010 | | |

## 第6页 · ALU 设计（代表运算细看）

- `add`/`sub`：基于 `addsub32`，顺带讲**溢出标志 `v`**（同号相加 / 异号相减后符号翻转）——它是第 5 节"溢出异常"的硬件来源
- `shift`：结构上的"异类"，操作数来自 `a[4:0]`（移位量），靠 `right`/`arith` 两个控制位区分逻辑/算术、左/右移
- `z = ~|r` 零标志，供 `beq`/`bne` 判分支

## 第7页 · 数据通路设计（全景框图）★

- 单周期 = **取指 → 译码 → 执行 → 访存 → 写回**，一个时钟周期内全部完成（无流水线）
- 关键部件：PC `dff32`、指令 ROM `scinstmem`、寄存器堆 `regfile`（2 读 1 写，r0 恒 0）、ALU 输入选择 `alua/alub`、数据 RAM `scdatamem`、写回三级 mux、下一 PC 选择 `pcsource`

🖼️ 配图（关键）：总数据通路框图 = `文档/datapath_p7.svg`（已生成，无需 Quartus；可在 draw.io 导入微调）

## 第8页 · 数据通路设计（沿代表指令点亮）★

在同一张框图上逐条高亮活跃路径：

- `add`：regfile 读 rs/rt → ALU(ADD) → 写回 rd（**最短主干**）
- `lw`：regfile 读 rs → ALU 算 `rs+符号扩展(imm)`（地址）→ 数据 RAM 读 → 写回 rt（**走完全程**）
- `beq`：regfile 读 rs/rt → ALU(SUB) 出 `z` → `pcsource` 选分支目标 `(PC+4)+offset`（**不写寄存器，改 PC**）
- `jal`：不读寄存器；**PC+4 经写回 mux 写入 r31**，同时 `pcsource` 选直接跳转目标（**既改 PC、又写返回地址**）

- 四条走完，ALU 运算 / 访存 / 分支改 PC / 写回 / 跳转 每条支路都点亮过

## 第10页 · 控制器设计（核心论点：9 运算 → 20 指令）★

- **复用**：多条指令落到同一种 ALU 运算（add/addi/lw/sw 都用 ADD；beq/bne 用 SUB 只取 `z`；and/andi 用 AND…）
- **区分**：靠正交控制信号（`aluimm/sext/m2reg/wmem/pcsource/shift`）裁剪同一 ALU 结果的去向
- 一句话：**指令数 = ALU 运算 × 数据通路其它控制信号的组合**

## 第11页 · 异常/中断设计（总体）★

- 3 内部异常：算术溢出、`syscall`、未实现指令；1 外部中断：`KEY[1]` 触发（单源、查询）
- 4 新指令：`mfc0`/`mtc0`/`syscall`/`eret`；3 个 CP0 寄存器：`Status(#12)`/`Cause(#13)`/`EPC(#14)`
- 统一入口 `EXC_BASE = 0x00000008`；`Cause.ExcCode` → 散转表 → 各处理程序；`eret` 返回
- `Status` 异常入口时左移 4 位（自动关中断、防嵌套）、`eret` 时右移 4 位（自动开中断）
