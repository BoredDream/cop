# 带异常/中断的单周期 MIPS CPU —— 实验设计报告

> 工程目录：`CPU/`（Quartus Prime 14.0，DE1-SoC，Cyclone V `5CSEMA5F31C6`）
> 顶层实体：`DE1_SOC_golden_top`　核心：`sccpu/sccpu_dataflow.v`
> 参考教材：《计算机原理与设计 Verilog HDL 版》第 6 章

---

## 摘要

本实验在 Terasic DE1-SoC 开发板上用 Verilog 实现了一个 **带异常/中断处理能力的单周期 MIPS CPU**。整体工作分三层递进：

1. **基础层**——搭建 Quartus + SignalTap 工具链，设计支持 9 种运算的 ALU；
2. **核心层**——在 ALU 之上组装经典单周期 MIPS 数据通路，支持 20 条基本指令，并修正了原始工程的两处数据通路缺陷；
3. **扩展层**——基于教材第 6 章，自行扩展异常/中断机制：新增 4 条指令、3 个 CP0 寄存器，支持 3 种内部异常 + 1 个外部中断，共 **24 条指令**。

最终通过 iverilog 仿真（`tb_intr.v`）与 DE1-SoC 板上数码管/LED 实测双重验证，CPU 能正确执行测试程序并依次响应算术溢出、系统调用、未实现指令、外部中断四类事件。

# 第一层　基础工具与运算单元

本层目标：打通开发/验证工具链，并实现 CPU 唯一的"计算核心"——ALU。它是后两层的地基。

## 1.1 实验环境与工具链

| 项目 | 内容 |
|---|---|
| 开发板 | Terasic DE1-SoC，FPGA 器件 Cyclone V `5CSEMA5F31C6` |
| 综合工具 | Quartus Prime 14.0（GUI 流程：Start Compilation → Programmer 烧录 `.sof`） |
| 在线调试 | SignalTap II 逻辑分析仪（`output_files/stp3.stp`/`stp4.stp`），抓取片内信号 |
| 语法快检 | `iverilog -g2012`（对不含兆功能的独立模块做编译检查与仿真） |
| 时钟 | 板载 50 MHz → PLL 分频到 10 MHz → 计数器再分出约 1 Hz 慢钟 |

工具链使用要点：

- **完全编译约 5 分钟**，SignalTap 配置被嵌入比特流，必须"先完全编译再烧录"才能抓信号。
- 顶层例化了 PLL（`pll_cpu`）与片内 RAM 兆功能，**iverilog 无法直接编译顶层**，只能在 Quartus 内综合；但 ALU、控制器、数据通路等纯逻辑模块可用 iverilog 离线验证。

## 1.2 ALU 设计（`sccpu/alu.v`）

ALU 是 32 位运算单元，由 4 位控制码 `aluc[3:0]` 选择运算，输出运算结果 `r[31:0]`、零标志 `z`、溢出标志 `v`。

### 1.2.1 端口

```verilog
module alu (a, b, aluc, r, z, v);
  input  [31:0] a, b;     // 两个 32 位操作数
  input  [3:0]  aluc;     // 运算选择码
  output [31:0] r;        // 运算结果
  output        z;        // 零标志：z = ~|r （结果全 0 时为 1）
  output        v;        // 溢出标志（仅 add/sub 类运算有效）
```

### 1.2.2 内部结构

ALU 不是一个大 case，而是**并行算出所有候选结果，再用多路选择器挑一个**，符合硬件并行特性：

```
        ┌─ addsub32 (a, b, aluc[2]) ──────► d_as   (加/减)
 a,b ───┤
        ├─ a&b / a|b ─ mux(aluc[2]) ──────► d_and_or (与/或)
        ├─ a^b / lui ─ mux(aluc[2]) ──────► d_xor_lui(异或/LUI)
        └─ shift (b, a[4:0], aluc[2], aluc[3]) ► d_sh (移位)
                                  │
        mux4x32( d_as, d_and_or, d_xor_lui, d_sh ; sel=aluc[1:0] ) ──► r
```

- **`aluc[1:0]`** 选大类（00=加减 / 01=与或 / 10=异或·LUI / 11=移位）；
- **`aluc[2]`** 在大类内二选一（如加/减、与/或），在移位中作"右移使能 `right`"；
- **`aluc[3]`** 在移位时作"算术移位使能 `arith`"。

依赖的子模块：`addsub32`（基于 32 位先行进位加法器 `cla32`，减法用 `b^{32{sub}}+sub` 求补实现）、`shift`（用 `<<` / `>>` / `>>>` 实现三种移位）、`mux4x32`（四选一）。

### 1.2.3 支持的 9 种运算与编码

| 助记符 | 运算 | `aluc[3:0]` | 实现路径 |
|---|---|:---:|---|
| ADD | `r = a + b` | `0000` | addsub32（加） |
| SUB | `r = a - b` | `0100` | addsub32（减） |
| AND | `r = a & b` | `0001` | 与门 |
| OR  | `r = a \| b` | `0101` | 或门 |
| XOR | `r = a ^ b` | `0010` | 异或门 |
| LUI | `r = {b[15:0],16'h0}` | `0110` | 立即数装高位 |
| SLL | `r = b << a[4:0]` | `0011` | 逻辑左移 |
| SRL | `r = b >> a[4:0]` | `0111` | 逻辑右移 |
| SRA | `r = $signed(b) >>> a[4:0]` | `1111` | 算术右移 |

> 注：移位量取自 `a[4:0]`，数据通路里 R 型移位指令把 `sa = inst[10:6]` 经 `alua` 选入 ALU 的 `a` 口。

### 1.2.4 标志位

- **零标志 `z = ~|r`**：结果按位或再取反，结果全 0 时 `z=1`。用于 `beq/bne` 分支判断（比较时 ALU 做 XOR，`z=1` 即两数相等）。
- **溢出标志 `v`**：仅在 `aluc[1:0]==00`（加减）时有意义。判定逻辑为"同号相加/异号相减后符号位翻转"：

  ```
  v = (~aluc[2] & ~a[31] & ~b[31] &  r[31])   // 正+正=负 → 溢出
    | (~aluc[2] &  a[31] &  b[31] & ~r[31])   // 负+负=正 → 溢出
    | ( aluc[2] & ~a[31] &  b[31] &  r[31])   // 正-负=负 → 溢出
    | ( aluc[2] &  a[31] & ~b[31] & ~r[31]);  // 负-正=正 → 溢出
  ```

  `v` 是第三层"算术溢出异常"的硬件来源，但控制器只在 `add/sub/addi` 时采信它（见 3.4）。

# 第二层　单周期 MIPS CPU 数据通路

本层目标：把 ALU、寄存器堆、存储器、各类多路选择器和 PC 逻辑组装成一个完整 CPU，每个时钟上升沿执行**一条**指令。讲义要求的 20 条指令在此实现。

## 2.1 总体架构

单周期 = "取指→译码→执行→访存→写回" 五个阶段在**一个时钟周期内全部完成**，无流水线、无延迟槽。顶层数据流（`sccpu/sccpu_dataflow.v`）：

```
   ┌──────────────────────── 控制单元 sccu_dataflow ───────────────────────┐
   │  op/rs/rd/func/z/v/sta/intr → 全部控制信号(wreg,aluc,pcsource,selpc...) │
   └───────────────────────────────────────────────────────────────────────┘
        │ 控制
   PC(dff32)─►scinstmem(指令ROM)─►inst─►译码
        │                                  │
        ▼                                  ▼
   [PC+4]/[分支]/[跳转]/[EPC]/[异常入口]   regfile(2读1写)
        │ next_pc                          │ra,data
        │                                  ▼
        │                       alua/alub 选择 ─► ALU ─► aluout
        │                                              │
        │                                  scdatamem(数据RAM) ◄─ sw写
        │                                              │ memout
        └──────────── 写回 res ◄─ mux(alu/mem/CP0/p4) ─┘
```

下面按"取指 → 译码 → 执行 → 访存 → 写回 → 取下条"的执行顺序逐段说明。

## 2.2 取指（Instruction Fetch）

- **PC 寄存器**：`dff32 ip(next_pc, clock, resetn, pc)`，下降沿异步复位归 0，上升沿载入 `next_pc`。
- **指令存储器**：`scinstmem imem(.a(pc), .inst(inst))`，64 字行为级 ROM，按 `rom[a[7:2]]` 取字（PC 是字节地址，低 2 位恒 0）。指令机器码直接以 `assign` 内联在 `.v` 里，便于版本控制。
- **PC+4**：`cla32 pcplus4(pc, 32'h4, 0, p4, ...)`，顺序执行的下一地址。

## 2.3 译码与控制单元（`sccpu/sccu_dataflow.v`）

控制单元是**纯组合逻辑**，输入指令字段，输出全部控制信号。

### 2.3.1 指令译码

先用布尔式从 `op`/`func` 还原出每条指令的"命中信号" `i_xxx`，例如：

```verilog
wire r_type = ~|op;                                   // op==000000 即 R 型
wire i_add  = r_type & func==6'b100000;               // (示意)
wire i_addi = ~op[5]&~op[4]&op[3]&~op[2]&~op[1]&~op[0];// op=001000
...
```

### 2.3.2 主要控制信号一览

| 信号 | 含义 | 置 1 的指令 |
|---|---|---|
| `wreg` | 写寄存器堆 | 所有有结果写回的指令（含 `jal`、`mfc0`） |
| `regrt` | 目标寄存器选 `rt`（否则 `rd`） | I 型 `addi/andi/ori/xori/lw/lui/mfc0` |
| `m2reg` | 写回数据取自存储器 | `lw` |
| `wmem` | 写数据存储器 | `sw` |
| `aluimm` | ALU 第二操作数取立即数 | `addi/andi/ori/xori/lw/sw/lui` |
| `shift` | ALU 第一操作数取移位量 `sa` | `sll/srl/sra` |
| `sext` | 立即数符号扩展（否则零扩展） | `addi/lw/sw/beq/bne` |
| `aluc[3:0]` | ALU 运算选择 | 由各指令对应运算决定（见 1.2.3） |
| `jal` | 链接（保存返回地址、写 r31） | `jal` |
| `pcsource[1:0]` | 正常流程下一 PC 来源 | 见 2.7 |

### 2.3.3 立即数与符号扩展

```verilog
wire e         = sext & inst[15];          // 需符号扩展且符号位为 1
wire [15:0] imm = {16{e}};                 // 扩展位
wire [31:0] immediate = {imm, inst[15:0]}; // 32 位立即数
```

## 2.4 通用寄存器堆（`sccpu/regfile.v`）

- 31×32 位寄存器（`register[1:31]`），**`r0` 恒为 0**（读端口对 `0` 号直接返回 0）；
- **2 个读端口**（`qa/qb`，组合读）+ **1 个写端口**（上升沿写，`we && wn!=0`）；
- 异步清零 `clrn`（接 `resetn`）。

## 2.5 执行（ALU 输入选择）

```verilog
mux2x32 alu_a(ra, sa, shift, alua);        // 移位指令用 sa，否则用 rs 值
mux2x32 alu_b(data, immediate, aluimm, alub);// I 型用立即数，否则用 rt 值
alu     al_unit(alua, alub, aluc, alu, zero, overflow);
```

`zero` 反馈给控制器做分支判断，`overflow` 反馈给控制器做溢出异常判断。

## 2.6 访存（`sccpu/scdatamem.v`）

```verilog
scdatamem dmem(.dataout(memout), .datain(data), .addr(aluout), .we(wmem), ...);
```

- 32 字行为级 RAM，按 `ram[addr[6:2]]` 寻址（`aluout` 作地址）；
- **组合读**（`assign dataout = ram[...]`）、**时钟写**（`lw` 取数 / `sw` 存数）；
- 初值含散转表与溢出测试数据（详见第三层 3.7）。

## 2.7 写回与 PC 更新

### 写回数据三级选择

```verilog
mux2x32 result  (alu, mem, m2reg, alu_mem);       // ① ALU 结果 vs 存储器数据
mux4x32 fromc0  (alu_mem, sta, cau, epc, mfc0, alu_mem_c0); // ② CP0 寄存器(第三层)
mux2x32 link    (alu_mem_c0, p4, jal, res);        // ③ jal 用 PC+4 作返回地址
assign  wn = jal ? 5'd31 : reg_dest;               // jal 强制写 r31
```

### 正常流程下一 PC（`pcsource`）

```verilog
mux4x32 nextpc(p4, adr, ra, jpc, pcsource, npc);
```

| `pcsource` | 来源 | 触发 |
|:---:|---|---|
| `00` | `p4`（PC+4） | 顺序执行 |
| `01` | `adr`（分支目标） | `beq`&相等 / `bne`&不等 |
| `10` | `ra`（寄存器值） | `jr` |
| `11` | `jpc`（直接跳转目标） | `j` / `jal` |

其中分支目标 `adr = (PC+4) + offset`，`offset = {sext(imm), inst[15:0], 2'b00}`（完整 16 位立即数符号扩展后左移 2）；`jpc = {p4[31:28], inst[25:0], 2'b00}`。

> `npc` 还不是最终 PC——第三层的异常/中断逻辑会用 `selpc` 在 `npc` 之上再做一次选择，得到真正写入 PC 寄存器的 `next_pc`（见 3.6）。

## 2.8 指令集（24 条）

| 类型 | 指令 | 功能 |
|---|---|---|
| **R 型 (9)** | `add` `sub` | 有符号加/减（可触发溢出异常） |
| | `and` `or` `xor` | 按位逻辑 |
| | `sll` `srl` `sra` | 逻辑左移/逻辑右移/算术右移 |
| | `jr` | 寄存器跳转 |
| **I 型 (9)** | `addi` | 立即数加（可触发溢出异常） |
| | `andi` `ori` `xori` | 立即数逻辑 |
| | `lw` `sw` | 取字/存字 |
| | `beq` `bne` | 相等/不等分支 |
| | `lui` | 装载立即数高 16 位 |
| **J 型 (2)** | `j` `jal` | 直接跳转 / 跳转并链接（写 r31） |
| **CP0/异常 (4)** | `mfc0` `mtc0` | 读/写 CP0 寄存器（第三层） |
| | `syscall` `eret` | 系统调用 / 异常返回（第三层） |

## 2.9 相对原始工程的两处修正（设计亮点）

`CPU/` 在 `实验3/` 基础上修复了两处数据通路缺陷，这是真正读懂代码的体现：

1. **`jal` 写回寄存器号**：旧版按常规 `reg_dest` 选择，导致返回地址未写入 `r31`。
   修正：`assign wn = jal ? 5'd31 : reg_dest;`
2. **分支偏移位宽**：旧版误用 `imm[13:0]`（14 位）拼接偏移，分支目标错误。
   修正：用完整 16 位立即数符号扩展 `offset = {imm, inst[15:0], 2'b00}`。
   同时控制器用 `branch_taken=(i_beq&z)|(i_bne&~z)` 修正了分支条件的 `pcsource[0]` 生成。

# 第三层　异常/中断扩展（本组亮点）

本层目标：在已能正确运行的单周期 CPU 之上，按教材第 6 章扩展异常/中断机制。**讲义并未要求这部分**，是本组的自定提升目标。

## 3.1 设计目标

| 类别 | 内容 |
|---|---|
| 内部异常（3 种） | 算术溢出、系统调用 `syscall`、未实现指令 |
| 外部中断（1 个） | 单源、查询方式，`KEY[1]` 触发 |
| 新增指令（4 条） | `mfc0` / `mtc0` / `syscall` / `eret` |
| 新增寄存器（3 个） | CP0 `Status(#12)` / `Cause(#13)` / `EPC(#14)` |
| 统一入口 | `EXC_BASE = 0x00000008` |

## 3.2 CP0 协处理器寄存器

用带写使能的 32 位寄存器 `dffe32` 实现，写使能分别为 `wsta/wcau/wepc`：

```verilog
dffe32 c0_Status (sta_in, clock, resetn, wsta, sta);  // #12
dffe32 c0_Cause  (cau_in, clock, resetn, wcau, cau);  // #13
dffe32 c0_EPC    (epc_in, clock, resetn, wepc, epc);  // #14
```

| 寄存器 | 号 | 作用 | 关键字段 |
|---|:---:|---|---|
| `Status` | 12 | 中断屏蔽与开关 | `IM[3:0]` = {溢出, 未实现, syscall, 外部中断} |
| `Cause` | 13 | 异常原因 | `ExcCode = Cause[3:2]` |
| `EPC` | 14 | 返回地址 | 异常时存 `pc`，中断时存 `npc` |

## 3.3 新增 4 条指令的译码

新增指令多数 `op = 0x10`（CP0 类，`c0_type`），靠 `rs` 字段（`op1`）区分：

| 指令 | 识别条件 | 语义 |
|---|---|---|
| `mfc0` | `op=0x10, rs=0` | `rt ← CP0[rd]`（读出 Status/Cause/EPC） |
| `mtc0` | `op=0x10, rs=4` | `CP0[rd] ← rt`（写入 CP0 寄存器） |
| `eret` | `op=0x10, rs=16, func=0x18` | 异常返回 |
| `syscall` | `op=0, func=0x0c` | 触发系统调用异常 |

控制器据此新增输出：`inta`(中断确认)、`exc`(异常发生)、`wsta/wcau/wepc`(CP0 写使能)、`mtc0`、`mfc0[1:0]`(读 CP0 选择)、`selpc[1:0]`(PC 选择)、`cause[31:0]`。

## 3.4 异常检测与优先级屏蔽

异常源经 `Status.IM[3:0]` 屏蔽后或合成总异常信号 `exc`：

```verilog
wire overflow = ov & (i_add | i_sub | i_addi);  // 仅算术指令采信溢出
wire int_int  = sta[0] & intr;                  // 外部中断（且未被屏蔽）
wire exc_sys  = sta[1] & i_syscall;
wire exc_uni  = sta[2] & unimplemented_inst;
wire exc_ovr  = sta[3] & overflow;
assign exc    = int_int | exc_sys | exc_uni | exc_ovr;
assign inta   = int_int;                        // 中断确认 → 点亮 LEDR[6]
```

- **未实现指令** `unimplemented_inst`：对全部 24 条合法指令命中信号取反——任何译不出的指令都判为未实现异常。
- **`Status.IM[3:0]` 充当总开关**：异常入口时 `Status<<4` 把使能位清空（自动关中断，防嵌套），`eret` 时 `Status>>4` 恢复（自动开中断）。

### 异常原因编码（写入 Cause[3:2]）

```verilog
wire ExcCode0 = i_syscall | overflow;            // Cause[2]
wire ExcCode1 = unimplemented_inst | overflow;   // Cause[3]
assign cause  = {28'h0, ExcCode1, ExcCode0, 2'b00};
```

| 事件 | `ExcCode` | 散转表偏移 |
|---|:---:|:---:|
| 外部中断 | `00` | 0x0 |
| `syscall` | `01` | 0x4 |
| 未实现指令 | `10` | 0x8 |
| 算术溢出 | `11` | 0xC |

## 3.5 异常/中断处理流程（硬件 + 软件协同）

**硬件**在检测到 `exc` 的当拍完成现场保存与跳转：

```
1. EPC  ← pc（异常）/ npc（中断）      // epc_l2 由 inta 选择
2. Cause.ExcCode ← 硬件生成的 cause
3. Status ← Status << 4                // 自动关中断
4. PC ← EXC_BASE (0x00000008)          // selpc=10
```

**软件**从统一入口 `0x08` 起，读 `Cause` 散转到各处理程序：

```asm
0x08  mfc0 r26, Cause      # 取异常原因
0x0c  andi r27, r26, 0xc   # 取 ExcCode<<2 作表内偏移
0x10  lw   r27, 0x20(r27)  # 查散转表（数据 RAM）
0x18  jr   r27             # 跳到对应处理程序
```

散转目标（数据 RAM 的散转表 → 各 entry）：外部中断→`0x30`，syscall→`0x3c`，未实现→`0x54`，溢出→`0x68`。

**返回**：`eret` 执行时 `PC ← EPC`（`selpc=01`）、`Status ← Status >> 4`（自动开中断）。
对 `syscall`/未实现/溢出，由于 EPC=触发指令地址，处理程序先做 `EPC ← EPC+4` 再 `eret`，从而跳过出错指令。

## 3.6 数据通路新增电路

### Status 输入选择（三选一）

```verilog
mux2x32 sta_l1 (sta_l1_a0, data, mtc0, sta_in);                       // mtc0 写软件值
mux2x32 sta_l2 ({4'h0,sta[31:4]}, {sta[27:0],4'h0}, exc, sta_l1_a0);  // exc 左移 / 否则右移
```

### EPC 输入选择

```verilog
mux2x32 epc_l1 (epc_l1_a0, data, mtc0, epc_in);    // mtc0 写软件值
mux2x32 epc_l2 (pc, npc, inta, epc_l1_a0);         // 中断存 npc，异常存 pc
```

### PC 最终选择（`selpc`，接管 `npc`）

```verilog
assign selpc[0] = i_eret;   assign selpc[1] = exc;
mux4x32 irq_pc(npc, epc, EXC_BASE, 32'h0, selpc, next_pc);
```

| `selpc` | `next_pc` | 场景 |
|:---:|---|---|
| `00` | `npc` | 正常流程 |
| `01` | `epc` | `eret` 返回 |
| `10` | `EXC_BASE` | 异常/中断入口 |

### mfc0 读 CP0 数据选择

```verilog
mux4x32 fromc0(alu_mem, sta, cau, epc, mfc0, alu_mem_c0);
// rd=12→Status, rd=13→Cause, rd=14→EPC
```

## 3.7 测试程序与数据（教材第 6 章）

`scinstmem.v`（64 字）按顺序触发 4 类事件，`scdatamem.v`（32 字）提供散转表与数据。

**散转表（数据 RAM）**

| 地址 | 值 | 含义 |
|---|---|---|
| 0x20 | 0x30 | 外部中断 entry |
| 0x24 | 0x3c | syscall entry |
| 0x28 | 0x54 | 未实现 entry |
| 0x2c | 0x68 | 溢出 entry |

**测试数据**：`0x48=2`、`0x4c=0x7fffffff`(max_int，二者相加溢出)；`0x50~0x5c` 为中断循环累加用的 4 个数。

**主程序关键节点**

| PC | 指令 | 作用 |
|---|---|---|
| 0x00 | `j start` | 复位后跳主程序 0x74 |
| 0x74 | `addi r8, r0, 0xf` | 准备开中断屏蔽位 |
| 0x78 | `mtc0 r8, Status` | `IM[3:0]=1111` 开全部中断 |
| 0x84 | `add r9,r9,r8` | **触发溢出异常** |
| 0x8c | `syscall` | **触发系统调用异常** |
| 0x94 | `div r9,r8` | 未实现指令，**触发未实现异常** |
| 0xa8 | `loop:` 累加循环 | 期间按 `KEY[1]` **触发外部中断** |
| 0xc0 | `j finish` | 结束死循环，`LEDR[7]` 点亮 |

# 第四层　验证方法与结果

## 4.1 离线仿真（iverilog）

`tb_intr.v` 测试平台串联控制器、数据通路、ALU、存储器等模块，自动跑测试程序，逐拍打印 PC/指令/CP0 寄存器，验证 3 异常 + 1 中断流程：

```bash
cd CPU/sccpu
iverilog -g2012 -o /tmp/tb_intr.vvp \
  tb_intr.v sccu_dataflow.v sccpu_dataflow.v alu.v addsub32.v shift.v \
  regfile.v cla32.v mux2x32.v mux4x32.v mux2x5.v dff32.v dffe32.v \
  scinstmem.v scdatamem.v
vvp /tmp/tb_intr.vvp
```

## 4.2 板上实测（核心判据）

慢钟（`SW[3]=1`）+ 显示 PC（`SW[1:0]=01`），按 `KEY[0]` 复位松开后观察 PC 逐拍变化：

```
000000 → 000074 → 000078 → 00007C → 000080 → 000084
        →(溢出) 000008 → 000068 →…→ 返回
        →(syscall) 000008 → 00003C →…
        →(未实现 div) 000008 → 000054 →…
        →(loop 中按 KEY[1]) 000008 → 000030 →…（LEDR[6] 亮一拍）
        →…→ 0000C0（LEDR[7] 亮）
```

**判据**：程序最终停在 `0xC0` 且 `LEDR[7]` 点亮 = 运行完全正确 ✅。按 `KEY[1]` 时 `LEDR[6]` 闪亮并进入 `0x30` 中断处理 = 中断机制正确。

## 4.3 SignalTap 在线抓取

顶层把 `pc/inst/aluout/memout/opcode/func/wmem` 引到带 `synthesis keep` 的 `debug_*` 网，配合 `stp3.stp`/`stp4.stp`，由 `KEY[0]` 触发抓取，可在 PC 上直接看波形，与 `scinstmem.v` 逐条对照。
